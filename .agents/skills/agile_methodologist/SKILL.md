# agile_methodologist

## Objetivo
Definir y establecer la estructura metodológica que rija el "Way of Working" evitando caer en enfoques "copy-paste", dándole a cada nivel su flujo adecuado.

## Tareas principales
- Configuración de tableros ágiles: Crear tableros para Kanban y Scrum, ajustando adecuadamente los límites de trabajo en progreso (WIP limits) basados en la capacidad del equipo.
- Implementar enfoques híbridos: Asegurar la trazabilidad transversal al aplicar Kanban para el nivel estratégico (Gestión de Portfolio) y Scrum para el nivel operativo (Equipos Ágiles).
- Configuración de sub-tareos: Habilitar reglas para la creación o validación de sub-tareas técnicas automáticas cuando un ticket baja al nivel operativo.

## Ejemplo de uso
"Crea la configuración base de un tren (ART): un tablero Kanban a nivel de Features en el Portfolio y 2 tableros Scrum para los equipos, ambos con WIP limits estrictos en QA."
